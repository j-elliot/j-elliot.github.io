"Task Manager V2"
28 November, 2020

I'm trying to get my shit together. I'm running a business. I have other project. I need to get over myself and become more efficient. I have tried a number of things in that respect. Some techniques work to a degree, others less so. Now I am trying to make progress using my professed profession. I'm writing a program to help me manage my life.
Now this project exists for a couple reasons. I want to be more organized and focused. I want to use my time more efficiently. The other goal here is learning SQL, and learning to get information in and out of a SQL database programmatically. A couple months ago I knew nothing about the technical workings of databases. I kind of got that it makes sense that a database is on a server, but I had no idea how to get, start, or run one. Now I do, at least with MariaDB. I have Python reaching out and accessing it on a local machine, and I designed a tree structure database and wrote code to navigate and manipulate it.
I created a task manangement system, with a root task as a project, and all the sub tasks as branches and leaves on the tree. It is very basic. I can navigate the tree, add tasks, add hours spent on the tasks, and mark tasks as complete. The development was very meta as the example project I used for testing was the project of building the task manager itself. It only runs on my local machine as the Pytho-SQL integration is hardcoded and specific. Once I fix that, I will put the code on github.
Programming is easy to see as boring, but this project wasn't for me. Sure, it fits the mold of plan, google, code, google, debug, and sleep. But it is exhilirating to learn a new thing, even if that thing is something lots of people have already learned. I don't know how to convey that excitement, though. I don't know how to bring my readers along on my journey. When I figure that out, I'll let you know.
